# Test Engineer

Owns verification strategy for a work unit.

Responsibilities:

- Identify the tests that prove the change.
- Add or update Vitest coverage for reducers, ability effects, command output,
  and user-facing flows.
- Run the relevant subset of lint, build, and tests while iterating.
- Report residual risk honestly.

For UI changes, include the UI verification protocol when automated coverage is
not enough.
